\section{Założenia środowiska testowego}
\label{sec:warunki_testowe}

Podczas weryfikacji inferencji kamerowej przyjęto następujące warunki:
\begin{itemize}
    \item równomierne oświetlenie bez silnych cieni,
    \item jednolite tło ograniczające fałszywe detekcje,
    \item dłoń ustawiona frontalnie względem kamery,
    \item wyłączone odbicie lustrzane obrazu.
\end{itemize}

Zmiana warunków oświetlenia lub geometrii kadru obniża jakość detekcji dłoni,
a tym samym stabilność predykcji.

\section{Analiza jakości inferencji w czasie rzeczywistym}
\label{sec:analiza_live}

W trybie \texttt{infer --source camera} system wykonuje predykcję dla każdej
klatki wejściowej. Jakość predykcji zależy od:
\begin{enumerate}
    \item poprawnej detekcji konturu dłoni przez MediaPipe,
    \item jakości kadrowania i skalowania do 28$\times$28,
    \item zgodności rozkładu danych wejściowych z rozkładem treningowym.
\end{enumerate}

Najczęstsze problemy obserwowane podczas demonstracji:
\begin{itemize}
    \item brak dostępu do kamery (\texttt{camera index} zajęty),
    \item słaba separacja dłoni od tła,
    \item częściowe wyjście dłoni poza obszar kadru.
\end{itemize}

\section{Ograniczenia obecnej wersji}
\label{sec:ograniczenia}

Wersja aktualna jest rozwiązaniem klasyfikacji pojedynczych klatek i nie
modeluje dynamiki ruchu. Oznacza to, że:
\begin{itemize}
    \item znaki wymagające ruchu (np. J, Z) nie są klasyfikowane,
    \item brak jest mechanizmu wygładzania predykcji po czasie,
    \item brak detekcji kontekstu sekwencji gestów.
\end{itemize}

Mimo tych ograniczeń projekt spełnia cel dydaktyczny i demonstracyjny:
spójny pipeline od danych tablicowych do inferencji online.
